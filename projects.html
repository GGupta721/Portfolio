Description:

Social anxiety and isolation have been at an all time high due to the recent pandemic. Many kids, teens, and adults already battling depression, and other mental issues are now facing an exponentially higher risk. To battle against this global struggle, you can safely meet with users in our online playground where Common interests like books, classes, tv shows etc. can be discussed in a room with other interested users.

System Design:

We used AWS Amplify to deploy standard HTML/JS/CSS front end.

The Chat Microservice uses AWS Websocket API to enable chat functionality between the multiple clients. We used two DynamoDB databases. One is keeping track of open socket connections for broadcasting purposes and the other one for the message history. We also used Lambda functions for adding/disconnecting clients, adding new messages to the databases. The lambda functions use Python.

The Profile Microservice uses Rest API to enable the storage and retrieval of user profile data. It uses two Lambda functions. One for setting user information on signup & another for etting user information for chat purposes and profile page. It also uses a DynamoDB database for storing user data that is not stored within AWS Cognito user pool. The lambda functions use Python.

AWS Cognito is used for creation of new users in the User Pool and providing authentication. It requires email-id and name for creating a new user and also uses email-id verification.

Technologies Used:

AWS Amplify was chosen to deploy the front-end static resources, as it is flexible and will facilitate our plan for further development in the next iteration. AWS Cognito was used to provide authentication and user services. It allows for easy integration with the app and makes a secure and scalable user directory. The web application was deployed as a serverless application using AWS Lambda for the compute service, and store or get necessary information from DynamoDB. We chose DynamoDB over Amazon Keyspaces because even though Amazon Keyspaces is more scalable, it has poor performance for inserting single records which is required throughout the app.
